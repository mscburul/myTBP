Package: de.geobe.util.statemachine

[Groovy] Class StateMachine<S extends Enum, E extends Enum>

    • de.geobe.util.statemachine.StateMachine

    • Implementation of a state machine with these features

      • States as enums,
      • Events as enums,
      • Transitions are triggered by a combination of current state and event,
      • they can be guarded by a Closure inhibiting their execution,
      • onExit activities executed when leaving a state,
      • internal activities triggered by an event without state change.
      • onEntry activities executed when entering a state,
      • transitions executing activities after leaving a state and before entering a following state. They can return a target state changing the following state.
      Authors:
      georg beier
      Parameters:
      S - enumertion Type for States. Limited by current implementation to 4095 values.
      E - enumeration type for events

        • void addEntryAction(S state, groovy.lang.Closure action)

          action - Closure that implements the activity. Must not have parameters.

        • void addGuardedTransition(S from, S to, E ev, groovy.lang.Closure guard, groovy.lang.Closure action = {})

          Add a transition to the state machine linking two states

          Parameters:
          from - the state, from where transition starts
          to - the state, where transition leads to. May be same as from state.
          ev - the event that triggers this transition
          guard - Closure that controls execution of the transition. Only when it returns (Groovy-) true, transition is executed
          action - Closure that is executed during the transition. It may return a different target state that overwrites to parameter. Thus branching transitions can be easily implemented. Closure may have parameters of type Object... .

        • void addTransition(S from, S to, E ev, groovy.lang.Closure action = {})

          Add a transition to the state machine linking two states. It is not guarded, i.e. its guard is always true.

          Parameters:
          from - the state, from where transition starts
          to - the state, where transition leads to. May be same as from state.
          ev - the event that triggers this transition
          action - (optional, default is do nothing) Closure that is executed during the transition. It may return a different target state that overwrites to parameter. Thus branching transitions can be easily implemented. Closure may have parameters of type Object...

        • S execute(E event, java.lang.Object... params)

          Actions are identified by currentState and incoming event. Execute first matching action with a guard delivering (Groovy-) true. If none is found, no action is executed and state stays unchanged. After execution, statemachine will be

          • in the following state as defined in addTransition method, if closure returns no object of type S
          • in the state returned by the closure.
          • If no following state is defined, statemachine will stay in currentState and no exitAction should have been executed.
          Parameters:
          event - triggering event
          params - optional parameter to Action.act. Caution, Action.act will receive an Object[] Array
          Returns:
          the current state after execution
